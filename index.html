三个决策对抗三个失效模式

每条规则都可追溯到真实事故，每个决策都对抗至少一个系统性失效模式。

三层模型
L1 始终加载 · L2 激活时≤300行 · L3 按需读取。控制 context 膨胀，让有效指令不被挤出。

⚡ 核心能力

工程化构建 Skill 所需的一切

从架构规划到质量门控——一个 Skill 覆盖全流程。

📐

三层模型架构

L1 触发词（~100词，始终加载）· L2 工作流（≤300行，激活时）· L3 参考文档（不限，按需读取）。保持 context 精简。

🚪

八阶段门控工作流

从理解需求到触发词优化——每个阶段有独立入口/出口条件，不满足不可跳过。

🛡️

20条反模式清单

结构膨胀类(5) · 流程缺陷类(5) · 规则质量类(4) · 一致性类(4) · 工程规范类(2)。每条可追溯到真实事故。

✅

23项质量审计清单

格式规范(9) · 结构健康(10) · 内容质量(7)。自动化扫描+人工确认，"目测没问题"不再是交付标准。

🔍

自动化审计脚本

一条命令全面扫描——绿/黄/红三色输出，23项逐条打勾。

🎯

触发词优化

扫禁止词、中英文触发词5+种、10+10准确率测试≥85%。漏触发意味着 Skill 等于不存在。

🧪

Eval 测试

有/无 Skill 对比，量化实际效果，不凭感觉。

⚠️

停滞循环检测

内置 stale_count 机制——量化信号逃出无限迭代陷阱。

📏

可验证性审查

将主观的"写高质量代码"转化为客观的"lint 通过无 error"。每条指令必须可 yes/no 判断。
